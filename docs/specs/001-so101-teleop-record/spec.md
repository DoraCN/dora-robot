## 0. 决策层级自检（SDD §0）

- 运行期是**确定性控制逻辑**（读主臂关节 → 下发从臂 → 按帧把数据交给 lerobot），属「代码层」，**运行期不需要 AI/Agent**。
- 策略学习/推理是**后续独立特性**，不在本特性内。
- 结论：SDD 在此作为**开发期**规范，而非在产品里引入运行期 AI。

---

## 1. Problem Statement（WHY）

操作者需要通过**物理搬动一只「主臂」**来**实时驱动远端「从臂」**完成操作任务；并把过程数据**交给 lerobot 录成可用于模仿学习训练的数据集**（**落盘由 lerobot 负责，非本项目**），以便后续训练/回放策略。目前没有这条「从遥操作到数据」的链路。

---

## 2. User Stories（谁在什么场景下用）

- 作为**操作者**，我搬动主臂（**含夹爪开合**）、从臂（含夹爪）实时跟随，使我能远程完成抓取/操作任务。
- 作为**操作者**，我能开始/结束一个回合（episode）；本项目把该回合的数据**交给 lerobot**（由其存为一条 episode）。
- 作为**操作者**，在结束回合时把它标为**成功**或**失败**；本项目据此**调用 lerobot**（成功→保存调用、失败/重录→丢弃调用），由 lerobot 决定落盘。
- 作为**标注者/操作者**，我能为本次录制提供**任务描述**，使数据可用于任务条件化训练。
- 作为**操作者**，链路断开或我按下急停时，从臂立即进入安全态、不乱动。
- 作为**操作者**，即使录制出错/变慢，遥操作控制**也不受影响**（录制是旁路、只丢不堵）。
- 作为 **ML 工程师**，我能用 lerobot **直接加载**录得的数据集来训练模仿策略。

## 3. Success Metrics（每条均可测试；数值已定稿）

| # | 指标 | 判定（可测试） |
|---|---|---|
| M1 | 跟随精度 | 在**约定参考轨迹**下（**逐关节顺序**做 ±30°、角速度 ≤ 30°/s 的往返），运动**沉降后**（主臂指令静止 ≥ 200 ms）测主从同关节误差：臂关节 **P95 ≤ 2°**；**夹爪舵机角误差 ≤ 5°** |
| M2 | 端到端命令时延 | 定义 `t0=主臂关节被采样时刻`、`t1=从臂将该指令施加到执行器时刻`：**`t1−t0` ≤ 50 ms（有线 LAN）**（**不含**舵机机械响应）。两机时钟同步测量，或以链路往返时间/2 近似（假设路径对称）。（WiFi/4G 等其它链路的时延预算待对应迭代再定） |
| M3 | 控制频率 | **在录制开启**的 ≥ 5 分钟会话中，控制环频率 **P5 ≥ 100 Hz**，且最小瞬时 ≥ **90 Hz** |
| M4 | 防弹射（上线一次性） | 进入实时跟随后**首个控制拍**，从臂各关节「目标−当前」步长 ≤ **2°** |
| M5 | 安全态 | 「安全态」= 从臂**停止跟随并保持当前关节位置**；若链路在 **T_drop = 2 s** 内未恢复则**卸力**。计时起点 = 事件时刻（急停按下 / 末次有效指令之后超过允许最大间隔，即本指标的 200 ms）。**急停(本地) ≤ 1 控制拍(~20 ms)** 进入保持；**链路丢失 → 保持 ≤ 200 ms**；全程**不重放过期指令** |
| M8 | 录制无侵入 | 相对「录制关闭」基线：录制开启时 **M3 频率降幅 < 5%**、**M2 时延增幅 < 10%** |
| M9 | 数据交付（**传**给 lerobot） | 本项目按约定**正确把数据交给 lerobot**：每帧含 `action`/`observation.state`（DoF=6）/非空 `task`，**30 Hz** 交付；并据回合结果**正确调用**（成功→保存调用、失败/重录→丢弃调用）。**落盘/数据集内容由 lerobot 负责，非本项目验收** |
| M10 | 单拍运动上限（持续安全） | 任意控制拍，从臂单关节指令变化量 ≤ **3°/拍（≈ 300°/s @ 100 Hz）** |
| M11 | 录制故障隔离 | 录制器崩溃/阻塞时，控制环**不中断、不降级**（M3 仍达标），遥操作继续 |

> 注：`fps = 30` 指本项目**交给 lerobot 的帧率**（从 100 Hz 控制流降采样；与后续加相机保持一致）；落盘后的行数等由 lerobot 决定（非本项目验收）。
> 编号留空 M6/M7（Rev 4 移出，见 §10）。

## 4. Acceptance Criteria（怎么验证）

- **AC1 ↔ M1**：按 M1 参考轨迹与沉降条件采样，臂关节误差 P95 ≤ 2°、夹爪 ≤ 5°。
- **AC2 ↔ M2**：注入主臂阶跃/正弦，按 `t0/t1` 定义测得 `t1−t0` ≤ 50 ms（有线 LAN）。
- **AC3 ↔ M3**：录制开启、≥ 5 分钟会话，控制环频率 P5 ≥ 100 Hz、最小 ≥ 90 Hz（日志佐证）。
- **AC4 ↔ M4**：冷启动 → 进入跟随首拍，各关节步长 ≤ 2°，无超调。
- **AC5 ↔ M5**：(a) 按急停 → ≤ 20 ms 进入保持；(b) 切断链路 → ≤ 200 ms 进入保持，2 s 未恢复 → 卸力；日志显示无过期重放。
- **AC8 ↔ M8**：录制开/关两组对照，M3 降幅 < 5%、M2 增幅 < 10%。
- **AC9 ↔ M9**：(a) **spy** 验证本项目以约定字段/30 Hz **正确把帧交给** lerobot 入口；(b) 成功→调用保存、失败/重录→调用丢弃，**调用正确**。（落盘结果由 lerobot 负责，**非本项目验收**）
- **AC10 ↔ M10**：注入主臂快速大幅运动，从臂逐拍步长被限在 ≤ 3°（不被弹射/超速）。
- **AC11 ↔ M11**：录制中**杀死录制进程** → 控制环 M3 不受影响、遥操作继续。

## 5. Non-Goals（本期明确**不做**，防止自由发挥）

- **数据落盘 / 数据集产出 / 可加载性**（v3 写入）：**lerobot 的功能，不属本项目**——不实现、不验收、不测；本项目只负责把数据**交给** lerobot。
- **摄像头 / 图像观测**：本期**延后**，只录本体关节（proprioception），不录视频。
- 双边力反馈。
- 异构从臂 / 逆运动学 / 笛卡尔模式。
- 策略**训练与推理**（本期仅采集传输）。
- 多臂 / 双臂。
- 视频编码性能调优（本期无视频）。
- **跨会话追加/合并**到同一数据集（本期：一次会话 = 一份数据交给 lerobot）。
- **自动**成功/失败判定（本期由操作者**手动**标记）。

## 6. Constraints & 前置条件（外部约束，**非**实现方案）

- 硬件：两台物理 **SO-101**（6 DoF，Feetech 总线舵机），一主一从，分处两台机器。
- 主臂**人手可反驱**（柔顺）；从臂**主动驱动**。
- 两臂经**网络链路**通信（当前 LAN；后续可能 WiFi/4G/5G）。
- **前置条件**：会话开始前，主从两臂已完成**机械置零/标定**，使同一关节序号语义一致（否则 M1 不成立）。
- 本项目须按 **lerobot 约定的接口/字段（v3）** 把数据交给 lerobot；**落盘与可加载性由 lerobot 负责（非本项目）**。
- **数据处理一律遵循 lerobot 约定**（字段、好坏数据处理：失败即丢弃），不自造；**不修改 lerobot 源码**（`lerobot/` 只读）。
- **任务标注**：每次录制须提供任务描述（自然语言），随数据交给 lerobot。
- 舵机**总线带宽有限**（限制控制频率上限）。
- 须符合项目 `constitution.md` 的**架构与模块边界约束**（三端解耦等；具体架构选型见 constitution/plan）。
